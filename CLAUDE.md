# dotfiles - Emacs + Nix home-manager

A vanilla Emacs config managed by a Nix home-manager flake. The Nix layer
(`flake.nix` + `home.nix`) pins the Emacs binary, CLI tools, and language
servers. Plain Elisp under `emacs/` is symlinked into `~/.config/emacs/` and is
never generated by Nix; elpaca pins the Elisp packages.

## Read first

- `docs/GETTING-STARTED.md` - daily launch, keybindings, the dev loop, escape
  hatches. Start here.
- `docs/HANDOFF.md` - status snapshot and deferred backlog, if present. This
  file is local-only (gitignored), so it exists only on the original machine.
- `docs/plans/` and `docs/specs/` - the as-built build recipe and design doc.

## The dev loop (how config changes take effect)

`~/.config/emacs` is read-only Nix store symlinks. Editing an installed file
does nothing. To change the config:

1. Edit the source file under `emacs/` (`early-init.el`, `init.el`, or
   `emacs/lisp/*.el`). Never edit under `~/.config/emacs`.
2. `cd ~/dotfiles && home-manager switch --flake .#"$(whoami)"`
3. Quit Emacs (`C-x C-c`) and relaunch (`ec`). For a quick in-session try,
   `M-x eval-buffer`, but commit + switch + relaunch is the real path.

For a CLI tool or language server, edit `home.nix` instead, then `switch`. For a
new Elisp package, add the `use-package` form, `switch`, then let elpaca install
on next launch.

## Conventions and traps

- **New-file trap:** a brand-new `emacs/lisp/*.el` is invisible to
  `home-manager switch` until it is `git add`ed/committed - Nix's
  `xdg.configFile` only includes git-tracked files. New module loop:
  create -> commit -> switch. Edits to tracked files: commit-then-switch as usual.
- **Launch via the `.app` bundle** (`ec`, or
  `open -a ~/.nix-profile/Applications/Emacs.app`). No Emacs daemon: a bare
  `emacs &` or a launchd daemon GUI frame gets no keyboard focus on macOS.
- **Machine-local files live outside the repo** in `~/.config/emacs-local/`
  (`local.el` for forge hosts, `llm-local.el` for the gptel backend) and are
  never committed. The committed config loads them with a NOERROR guard.
- **`projects.el`, not `project.el`** - the module name avoids a recursive-require
  clash with the built-in `project`.
- **elpaca loads packages asynchronously** after `after-init-hook`. Code that must
  run after all packages are loaded uses `elpaca-after-init-hook`. See the
  run-now-or-defer pattern (`dabd/easysession-start` in `editing.el`) and the
  compat/transient notes in `docs/HANDOFF.md` before debugging package-load order.

## This repo is public-bound

The repo is intended to go public, and its history was rewritten to scrub
identifiers. Keep committed files free of work-specific names, secrets, and the
mechanical writing tells (em dashes, smart quotes, filler). Secrets stay in
machine-local files or 1Password references, never in tracked files.

## Git

- `git` is the system binary, not Nix-managed, to preserve enterprise credential
  helpers.
- Force-push to `main` is blocked by a safety hook and cannot be bypassed from
  inside this harness; run it in a separate terminal when a history rewrite needs it.
